# 메뉴 이미지

각 카페의 메뉴별 이미지를 이 폴더에 넣으면 드롭다운에 자동으로 표시됩니다.
이미지가 없으면 카테고리 이모지(☕🥛🧊🥤🍵)로 자동 폴백됩니다.

## 디렉토리

```
images/
  starbucks/     스타벅스 메뉴
  ediya/         이디야커피 메뉴
  mega/          메가MGC커피 메뉴
  compose/       컴포즈커피 메뉴
  twosome/       투썸플레이스 메뉴
  hollys/        할리스커피 메뉴
  paik/          빽다방 메뉴
  other/         기타 카페 메뉴
```

## 파일 이름 규칙

**한글 메뉴명 그대로, 공백은 `_`로.**

메뉴 원본 이름 | 파일명
--- | ---
`카페 아메리카노` | `카페_아메리카노.webp`
`바닐라 라떼` | `바닐라_라떼.webp`
`바삭 피스타치오 바닐라 크림 콜드 브루` | `바삭_피스타치오_바닐라_크림_콜드_브루.webp`
`자몽 허니 블랙 티` | `자몽_허니_블랙_티.webp`

코드가 `name.replace(/\s+/g, '_')` 로 파일명을 만들므로 이 규칙을 지켜야 매칭됩니다.

## 권장 스펙

- **포맷**: WebP (가볍고 iOS Safari도 지원)
- **크기**: 400 × 400 정사각형
- **배경**: 베이지 / 크림톤 (스타벅스 앱 장바구니 레퍼런스)
- **컨텐츠**: 메뉴 한 잔만 중앙 배치, 부차적 소품 최소화

## AI 생성 팁

프롬프트 예시:
> "A glass cup of iced 카페 아메리카노 on warm cream background, centered, product photo style, no text, 1:1"

배경 통일성을 위해:
- 모든 이미지에 같은 배경 팔레트 사용 (`#fdf5e6` 톤)
- 같은 조명/앵글
- 같은 렌더링 스타일 (realistic photo / illustrated 중 하나로 통일)

## 점진적 추가

- 처음엔 인기 메뉴 10~20개만 생성 → 드롭인
- 이모지 폴백 덕분에 UI 는 항상 정상 동작
- 시간 되는 대로 나머지 채움

## 메뉴 목록 참조

메뉴 이름은 `index.html` 의 `CAFE_MENUS` 와 `COMMON_MENUS` 에서 확인.
새 메뉴 추가 시 `addedAt` 필드에 `'YYYY-MM-DD'` 로 날짜 기록 → 14일 이내면 자동으로 🆕 NEW 뱃지가 붙음.
